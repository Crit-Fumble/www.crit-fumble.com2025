# FumbleBot Privacy Policy

**Last Updated:** 2025-01-27

## 1. Introduction

This Privacy Policy explains how FumbleBot ("we", "our", or "the bot") collects, uses, stores, and protects your information when you use our Discord bot service.

**Important:** By using FumbleBot, you agree to this Privacy Policy. If you do not agree, please do not use the bot.

## 2. Information We Collect

### 2.1 Automatically Collected Data

When you interact with FumbleBot, we automatically collect:

- **Discord User ID** - Your unique Discord identifier
- **Discord Server (Guild) ID** - The server where the bot is used
- **Discord Channel ID** - The channel where commands are used
- **Username and Discriminator** - Your Discord display name
- **Message Content** - Only when directly using bot commands
- **Voice Channel Metadata** - When using voice features (channel ID, connection time)
- **Timestamps** - When you use commands
- **Command Usage Statistics** - What commands you use and how often

### 2.2 User-Provided Data

When you explicitly provide information to FumbleBot:

- **Campaign Data** - Character names, campaign settings, session notes
- **Custom Commands** - Any custom commands you create
- **Preferences** - Settings you configure (notification preferences, etc.)
- **Foundry VTT Data** - World data, assets, and configurations if linked
- **RpgAssets** - Content you upload or reference (images, sounds, etc.)

### 2.3 AI Interaction Data

When you use AI-powered features:

- **Conversation History** - Recent messages for context (stored temporarily)
- **AI Prompts** - Commands and queries you send to the AI
- **Generated Content** - AI responses and generated materials

### 2.4 Technical Data

For system performance and debugging:

- **Error Logs** - Technical errors and stack traces
- **Performance Metrics** - Response times, resource usage
- **API Usage** - Calls to OpenAI, Anthropic, and Discord APIs

## 3. How We Use Your Information

### 3.1 Provide Services

- Analyze usage patterns to improve features
- Fix bugs and technical issues
- Optimize performance and response times
- Develop new features based on user needs

### 3.3 Communication
- Send important service announcements
- Notify about updates or maintenance
- Respond to support requests

### 3.4 Legal Compliance
- Comply with legal obligations
- Enforce our Terms of Service
- Protect against abuse and fraud
